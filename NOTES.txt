  2. Found all 30 drawing runs across 19 paragraphs
     - Page 1 shapes: paragraphs 10, 13, 29, 32, 34
     - Page 2 shapes: paragraphs 44, 48, 50, 52, 54, 56, 60,
                      65, 70, 72, 74, 78, 80, 88
     - Page 3 shapes: paragraph 91

     - Page 1 shapes: kept original X/Y coordinates
     - Page 2 shapes: shifted right by 595.35pt (one A4 width)
     - Page 3 shapes: placed on A3 page 2, same X/Y
  4. Updated both DrawingML (wp:anchor positionH/positionV)
     and VML fallback (v:shape/v:group margin-left/margin-top)
     to relativeFrom="page" with absolute coordinates
  5. Built new document body:
     - Para 1: all 29 page-1 shapes anchored here
     - Para 2: page break (creates A3 page 2)
     - Para 3: the 1 page-3 shape anchored here
  6. Set sectPr to A3 landscape (23811 x 16838 twips),
     margins ~1mm on all sides
  7. Wrote new document.xml back into a copy of the source .docx

  Click any text field (name, date, grade, city, etc.)
  and type directly - all are live editable text boxes.

  училишна_стручна_матура_А3.docx
  - A3 landscape (420mm x 297mm)
  - 2 pages
  - All 26 shapes preserved and fully editable

LAYOUT (same scheme as document 1):
  A3 Page 1 LEFT  = source page 1  (school header / student data)
  A3 Page 1 RIGHT = source page 2  (grades / school info)
  A3 Page 2 LEFT  = source page 3  (ministry stamp / verification text)
  A3 Page 2 RIGHT = empty

HOW IT WAS BUILT (create_a3_strucna.py):
  1. Used Word COM (win32com) to dump all 26 shapes: source page,
     RelativeHorizontalPosition/RelativeVerticalPosition, Left/Top.
  2. Parsed document.xml with lxml to list all 26 anchored drawing
     runs in true document order, extracting each textbox's text.
  3. Matched XML-order shapes to COM shape data by exact text
     (2 pairs of duplicate text were disambiguated by spatial
     position - e.g. two shapes both reading "Битола").
     - Page 1 shapes: paragraphs 9, 12, 25 (11 shapes)
     - Page 2 shapes: paragraphs 37, 38, 44, 50, 52, 57, 64, 66,
                      72, 74, 82 (13 shapes)
     - Page 3 shapes: paragraphs 126, 128 (2 shapes)
  4. Computed absolute page coordinates (adding page margins where
     RelativeHorizontalPosition/Vertical = margin-relative), then:
     - Page 1 shapes: kept original X/Y
     - Page 2 shapes: shifted right by 595.35pt (one A4 width)
     - Page 3 shapes: placed on A3 page 2, same X/Y
  5. Updated DrawingML (wp:anchor) and VML fallback to
     relativeFrom="page" with absolute coordinates.
  6. Built new document body (same 4-paragraph structure as doc 1)
     and A3 landscape sectPr.
  7. Verified output via Word COM: 2 pages, 26 shapes, correct
     per-shape page/position.

TO EDIT DATA:
  Open училишна_стручна_матура_А3.docx in Word.
  Click any text field and type directly - all are live editable
  text boxes.

FILES:
  create_a3_strucna.py          - Python script that built this file
  училишна_стручна_матура_А3.docx - Final output (USE THIS)
