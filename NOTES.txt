DIPLOMA A3 LAYOUT - WHAT WAS DONE

SOURCE FILE:
  "New folder\диплома државна матура (1).docx"
  - 3-page A4 portrait document
  - 30 floating shapes (text boxes + grouped border lines)
  - Uses both DrawingML and VML formats internally

OUTPUT FILE:
  диплома_А3_v2.docx
  - A3 landscape (420mm x 297mm)
  - 2 pages
  - All 30 shapes preserved and fully editable

LAYOUT:
  A3 Page 1 LEFT  = source page 1  (student personal data)
  A3 Page 1 RIGHT = source page 2  (grades / school info)
  A3 Page 2 LEFT  = source page 3  (stamp / verification text)
  A3 Page 2 RIGHT = empty

HOW IT WAS BUILT (create_a3_xml.py):
  1. Read the .docx as a ZIP file and parsed document.xml with lxml
  2. Found all 30 drawing runs across 19 paragraphs
     - Page 1 shapes: paragraphs 10, 13, 29, 32, 34
     - Page 2 shapes: paragraphs 44, 48, 50, 52, 54, 56, 60,
                      65, 70, 72, 74, 78, 80, 88
     - Page 3 shapes: paragraph 91
  3. For each shape, applied pre-verified absolute positions:
     - Page 1 shapes: kept original X/Y coordinates
     - Page 2 shapes: shifted right by 595.35pt (one A4 width)
     - Page 3 shapes: placed on A3 page 2, same X/Y
  4. Updated both DrawingML (wp:anchor positionH/positionV)
     and VML fallback (v:shape/v:group margin-left/margin-top)
     to relativeFrom="page" with absolute coordinates
  5. Built new document body:
     - Para 1: all 29 page-1 shapes anchored here
     - Para 2: page break (creates A3 page 2)
     - Para 3: the 1 page-3 shape anchored here
  6. Set sectPr to A3 landscape (23811 x 16838 twips),
     margins ~1mm on all sides
  7. Wrote new document.xml back into a copy of the source .docx

TO EDIT DATA:
  Open диплома_А3_v2.docx in Word.
  Click any text field (name, date, grade, city, etc.)
  and type directly - all are live editable text boxes.

FILES IN THIS FOLDER:
  create_a3_xml.py       - Python script that built the A3 file
  диплома_А3_v2.docx     - Final output (USE THIS)
  диплома_А3.docx        - Earlier attempt (copy-paste method, broken)

HOW IT WAS BUILT (create_a3_strucna.py):

  3. Matched XML-order shapes to COM shape data by exact text
     (2 pairs of duplicate text were disambiguated by spatial
     position - e.g. two shapes both reading "Битола").
     - Page 1 shapes: paragraphs 9, 12, 25 (11 shapes)
     - Page 2 shapes: paragraphs 37, 38, 44, 50, 52, 57, 64, 66,
                      72, 74, 82 (13 shapes)
     - Page 3 shapes: paragraphs 126, 128 (2 shapes)
  4. Computed absolute page coordinates (adding page margins where
     RelativeHorizontalPosition/Vertical = margin-relative), then:
     - Page 1 shapes: kept original X/Y
     - Page 2 shapes: shifted right by 595.35pt (one A4 width)
     - Page 3 shapes: placed on A3 page 2, same X/Y
  5. Updated DrawingML (wp:anchor) and VML fallback to
     relativeFrom="page" with absolute coordinates.
  6. Built new document body (same 4-paragraph structure as doc 1)
     and A3 landscape sectPr.
  7. Verified output via Word COM: 2 pages, 26 shapes, correct
     per-shape page/position.

TO EDIT DATA:

DOCUMENT 3: ДРЖАВНА СТРУЧНА МАТУРА (drzavna strucna.docx) - A3 LAYOUT
================================================================

SOURCE FILE:
  "New folder\drzavna strucna.docx"
  - 4-page A4 portrait document, but structurally very different from
    documents 1 and 2: almost entirely NORMAL FLOWING TEXT + one
    grades table, split across 3 pre-existing Word sections (one per
    source page), plus 2 stray trailing paragraphs ("ДИПЛОМА" / "ЗА
    ДРЖАВНА СТРУЧНА МАТУРА") that started an unwanted 4th page.
  - Only ONE floating shape in the whole document: the "Директор"
    signature text box (RelativeHorizontalPosition=Page,
    RelativeVerticalPosition=Paragraph), anchored on source page 2.
  - Per user decision: the stray 4th page is dropped; only the first
    3 pages are used, same scheme as documents 1 and 2.

OUTPUT FILE:
  диплома_државна_стручна_матура_А3.docx
  - A3 landscape (420mm x 297mm), 2 pages, the 1 shape preserved and
    fully editable; all other content is normal live text/table
    (never was a shape, so nothing to preserve there either).

LAYOUT (same scheme as documents 1 and 2):
  A3 Page 1 LEFT  = source page 1 (personal data)
  A3 Page 1 RIGHT = source page 2 (grades table + signatures)
  A3 Page 2 LEFT  = source page 3 (verification footer)
  A3 Page 2 RIGHT = empty

HOW IT WAS BUILT (create_a3_drzavna_strucna.py) - a DIFFERENT technique
than documents 1/2, since there are no per-field floating text boxes
to reposition, just flowing text across 3 sections:
  1. Found the document already had 3 sections (one sectPr per source
     page, each carrying that page's own pgMar) plus 2 stray trailing
     paragraphs after the 3rd section's content - confirmed via
     Word COM (doc.Paragraphs / doc.Sections) and lxml body-children
     dump (paragraph indices 12 and 27 carry sectPr; indices 34-35 are
     the stray paragraphs; index 36 is the body-final sectPr).
  2. Source page 1 + page 2 are merged into ONE new section using a
     genuine unequal-width w:cols 2-column layout (needs BOTH
     w:num="2" AND w:equalWidth="0" - Word silently treats the section
     as single-column and turns column breaks into page breaks if
     w:num is omitted, even with 2 <w:col> children present and
     equalWidth=0 - this cost one failed verify-via-COM round trip).
     Column widths/gutter were computed from each source page's own
     original margins (page1 left/right vs page2 left/right, which
     differ slightly) so each column reproduces that page's original
     usable text width exactly. This is algebraically identical to
     the simpler "shift column 2 by +595pt (one A4 width)" rule from
     documents 1/2 - verified by hand before implementing.
  3. A w:br type="column" was inserted as the first run of source
     page 2's first paragraph to force it into column 2.
  4. Source page 1's top margin (1518 twips) is smaller than source
     page 2's (2574 twips); since a section has only one top margin,
     the smaller was used for the merged section, and the 1056-twip
     difference was added as extra "space before" on page 2's first
     paragraph - reproducing page 2's original vertical start without
     touching the shape's Y at all.
  5. The "Директор" shape's Y is RelativeVerticalPosition=Paragraph,
     so it automatically rides along with its anchor paragraph's
     reflow (no Y edit needed). Its X (RelativeHorizontalPosition=
     Page) was shifted +595pt (=7,556,500 EMU) in both the DrawingML
     wp:anchor posOffset and the VML v:shape style margin-left.
  6. Source page 3 was left on its own final section, just widened to
     A3 with a larger right margin (left margin and all content
     unchanged) so it keeps its original usable text width and only
     occupies the left half of the sheet - no shape work needed there
     since page 3 has none.
  7. The 2 stray trailing paragraphs were deleted from the body.
  8. Verified via Word COM: 2 pages, 1 shape on page 1 at the shifted
     X, section 1 TextColumns.Count==2 with the expected widths/gutter,
     then rendered to PDF (via ExportAsFixedFormat + PyMuPDF) and
     visually confirmed both A3 sheets match the intended layout.
